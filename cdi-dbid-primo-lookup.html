CDI DBID → Primo Results Lookup

Before you start

This tool helps you view records for CDI collections marked “Link in Record.” Paste the DBID from the collection’s CDI tab in Alma and open the matching records in LibrarySearch.

If the collection is set to Search Rights in CDI: Subscription, sign in to LibrarySearch first or the search may return no results.

How to check the results
  1. In Alma, open the collection’s CDI tab and copy the DBID. If more than one DBID is listed (e.g. IAO,AHNUU), copy them all, commas included.
  2. Paste above and choose Open Primo results. The search runs as DBID,contains,JFNAL — the word “DBID” is Primo’s search field name and never changes; only your collection code after “contains” does. Multiple DBIDs are combined as DBID,contains,(IAO OR AHNUU).
  3. Don’t expect the result count to match the Number of records in the collection’s CDI settings — Ex Libris notes that DBID searches on large collections return partial results (a server time-out truncates them), and Primo merges duplicate records and filters by our subscriptions. What matters: results exist, and spot-checked records are actually from this collection.
  4. Zero results for a collection that should have many is the signal worth investigating — first sign in to LibrarySearch (or sign out and back in) and try again.

Special cases

  • Some collections (Web of Science, ABI/INFORM, DOAJ, and others) are grouped in CDI and only findable by group name, such as GROUPED_DOAJ — their individual DBIDs return nothing.
  • DBIDs are case-sensitive. The tool capitalizes automatically, which matches standard DBIDs (including group names).

Source: Ex Libris: CDI Tips and Tricks (DBID search syntax, multiple-DBID collections, group names, and the result-count time-out).
